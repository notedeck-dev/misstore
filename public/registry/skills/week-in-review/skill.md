# 週次振り返り

ユーザーが「最近の私を振り返って」「先週どうだった?」「週次レビューして」のような依頼をしたとき、自分のノートを大量に読み込んで活動を要約する。

## 手順

1. `account.current` で active アカウントの `userId` を取得
2. `notes.user` を `userId` 指定で 100 件取得
3. 取得したノート群の最古の `createdAt` がユーザー指定期間の開始日 (デフォルト: 過去 7 日) より新しければ、最後のノートの `id` を `untilId` に渡してさらに 100 件取得。最大 5 回繰り返す (合計 500 件まで)
4. 期間外のノートは捨てて分析対象を確定する
5. 以下の項目で分析結果を出力する

## 分析項目

- **主要トピック** — 頻出キーワード・言及対象を 5〜10 個。漢字熟語/固有名詞優先
- **投稿時間帯の偏り** — `createdAt` から朝 (5-11)/昼 (11-17)/夜 (17-翌 5) の比率を計算
- **長めの内省 Top 3** — `text` の長さが上位の 3 件 (CW で隠れていないもの)。冒頭 1 行ずつ抜粋
- **トーン傾向** — 前向き / 落ち着き / 興奮 / 愚痴 / 雑談 のどれが多いか主観で判定
- **興味の変化** — 期間の前半と後半で話題がどう移ったか 1 段落

## 出力フォーマット

Markdown 見出し付き。
- 冒頭に 2〜3 行の総評 (ノート数・期間・主な印象)
- 上記 5 項目をセクション化
- 末尾に「来週の関心事として伸びそうなテーマ」を 1 つだけ予想

## 制約と注意

- ノートの projection は `{ id, userId, username, text, createdAt }` のみ。`renoteCount` や `reactions` は取れないので「人気投稿ランキング」は出さない
- CW 付きノートは `text` が `[CW: 理由]` に置換されている。中身は推測せず、CW 理由だけで概観する
- ノート数が 30 件未満なら「サンプル不足」と素直に告げて分析を打ち切る
- 期間内ノートを 1 つも取得できなければ「期間内に投稿が見当たりません」とだけ返す
- リプライ相手の名前は伏せないが、誹謗中傷を含むやり取りは中立的に圧縮する
