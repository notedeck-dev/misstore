# マインドフルパルス

ユーザー本人の Misskey での過ごし方を、本人にだけ見せるレポートとして定期的に返す HEARTBEAT スキル。エンゲージメント最大化の **逆** をやる: 「もう少し離れていいよ」という気づきを設計の核に置く。

## 起動条件

HEARTBEAT が発火したとき。観測ウィンドウは「直近 7 日」を基本とし、HEARTBEAT 間隔がそれより長ければ「前回 HEARTBEAT 以降」に拡張する。

## 観測項目

NoteDeck 側が記録している想定の利用統計と、Misskey API から取れる範囲のデータを組み合わせる:

- **滞在時間** — NoteDeck が記録する「アプリにフォーカスがあった秒数」(取れない場合はスキップ)
- **セッション数** — 起動回数 / 日 (同上)
- **投稿頻度** — `notes/user` で自分の userId を期間指定で取得し件数集計
- **時間帯分布** — 投稿の `createdAt` を朝 (5-11) / 昼 (11-17) / 夜 (17-23) / 深夜 (23-翌 5) に振り分け、特に **深夜** 比率を見る
- **通知未読** — `i/notifications` で取得 (取れる範囲で。鍵やレート制限で部分的でも可)
- **連続使用日数** — N 日連続で使っているか (DAU streak)

## 「気づき」設計の原則

レポートは数字を並べるだけでなく、ユーザー本人にだけ届く 1〜2 行の所見を末尾に入れる。原則:

- **滞在時間を伸ばさせる方向の助言は禁止**。「もっと投稿しよう」「フォロー候補です」「未読を消化しましょう」のような促しは出さない
- **比較で煽らない**。他人と比べる、過去の自分の最高値と比べて「下がってますね」と煽る、はしない
- **観察と提案のみ**。例: 「深夜投稿が 60% を超えています。睡眠リズムに余裕があれば、夜の 1 時間オフラインを試してみるのもよさそう」
- **休むことを肯定する**。投稿・滞在が減っている場合、それを「離脱」「停滞」と呼ばずに「静かな週」と呼ぶ
- **連続日数を強調しない**。streak は数字としては出してよいが「途切れさせるな」のフレーミングは禁止

## 出力フォーマット

```
## マインドフルパルス ({期間})

- 滞在: {N} 時間 {M} 分 / セッション {N} 回
- 投稿: {N} 件 (うち深夜帯 {XX}%)
- 通知未読: {N} 件
- 連続使用: {N} 日

### 気づき
{1〜2 行の観察。押し付けない}
```

データが取れない項目は行ごと省略する (「-」や 0 で埋めない)。

## 制約と注意

- このスキルの出力は **本人のチャット欄にだけ** 表示される想定。投稿として外に出さない
- 「ヘルシー」「アンヘルシー」のラベリングをしない。日が当たる時間も夜も、それぞれの過ごし方として扱う
- 体調・メンタルに関わる断定的な言及をしない (「燃え尽きてますね」「うつかも」など)。観察事実のみ述べ、専門家への相談を促す表現も自発的には出さない
- 数値が前週比で大きく変動していても、原因を勝手に推測しない (「仕事が忙しかったのかも」など)
- 利用ログ・通知データへのアクセスができない環境では「観察可能なデータが少ないため、今回のレポートはスキップしました」の 1 行で終える
- 月 1 回程度、長めの振り返り (1 ヶ月分のサマリ) を出してもよいが、これも上記の原則を守る
