# 会話相手の文脈

ユーザーが「この人最近どんなテンション?」「リプライ書く前に空気感確認したい」と頼んだとき、対象ユーザーの直近 50 件のノートを読んで会話前のコンテキストを 30 秒で把握できる形で返す。

## 手順

1. 対象ユーザーを特定する。優先順位:
   - ユーザーが明示的に `@user@host` を渡したとき → `user.lookup` で `id` 解決
   - `<currentColumn>` が user カラムなら、そのカラムの対象 user を使う
   - `<visibleNotes>` の中で起点となるノートが指定されたら、その `userId` を使う
2. `notes.user` を `id` 指定で 50 件取得 (1 ターンで十分。多すぎるとノイズ)
3. 取得直後の最新 1 件の `createdAt` をチェック — 24 時間以内に活動があるか判定
4. 以下の項目で「リプライ前ブリーフィング」を生成

## 分析項目 (リプライ書く前に役立つ情報のみ)

- **直近の活動** — 最新ノートが何時間前か。3 日以上空いていれば「最近は静か」と告げる
- **いま話している話題** — 直近 10 件の話題を 1〜2 行
- **テンション** — 直近 10 件のトーン (前向き / 落ち着き / 疲れ / イライラ / 雑談)
- **NG ポイント候補** — 過去 50 件で繰り返し愚痴っている話題があれば 1〜3 個 (踏まないように)
- **リプライ向け話題のフック** — 直近のノートで「他人が触れて嬉しそうな話題」を 1〜2 個

## 出力フォーマット

短めの Markdown。長文は嫌われる前提で **A4 半分以下** を目指す。
- 1 行サマリ (例: 「直近 6 時間アクティブ、ゲーム話題で機嫌よさそう」)
- 上記 5 項目を箇条書き (各 1〜2 行)
- 末尾に「リプライするなら」の一言提案 (オプション)

## 制約と注意

- 50 件未満しか取れなかったら「サンプル不足」と注記して分析は続行
- `notes.user` の projection に reaction 数は無いので「反応大きい話題」は出さない
- CW 付きノートは本文不可視。`[CW: 理由]` だけで「センシティブ系の話題」程度の判定に留める
- 鍵アカウントだと取得できないことがある。空配列なら「公開ノートが取得できませんでした」とだけ返す
- 「この人は〇〇な性格」という人格判定は避ける。あくまで直近の発言から読み取れる **状態** を述べる
- ユーザーがリプライ文の下書きまで頼んだ場合のみ草稿を提示。頼まれていないのに提案しない
