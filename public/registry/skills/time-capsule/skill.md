# タイムカプセル

Misskey 上の自分のノートから **「1 年前の今日」「2 年前の今日」… の投稿** を掘り出して、1 日 1 回だけそっと差し出す HEARTBEAT スキル。Twitter Memories の Misskey 版。

## 起動条件

HEARTBEAT が発火したとき。ただし「1 日 1 回」程度の頻度を想定。HEARTBEAT が短い間隔で何度も呼ばれても、同日内で複数回出力しない (`updatedAt` を見て当日既に出していたら静かに skip と告げる)。

## 手順

1. `account.current` で自分の `userId` を取得
2. 「今日の月日」を確定 (例: 5/2)。タイムゾーンは `Intl.DateTimeFormat().resolvedOptions().timeZone` で取れた値、無ければ Asia/Tokyo
3. N 年前 (N = 1, 2, 3, ...) の同じ月日について、`notes/show` ではなく `notes/user` を `untilDate` / `sinceDate` (UNIX ms) を駆使して取得
   - `sinceDate = N 年前の 0:00 (ローカル)`
   - `untilDate = N 年前の 23:59`
   - `userId` は自分。最大 100 件
4. N = 1 から順に試して、見つかった分だけ束ねる。N の上限は 5 (5 年分まで遡る)

## 選別

該当ノートが多い (10 件超) なら、以下の優先度で 3 件だけピックアップ:

1. テキスト長 100 文字以上の内省的なノート
2. リアクション数が多い (取れなければスキップ)
3. リプライではない自発投稿
4. CW 付きでないもの

## 出力フォーマット

```
## タイムカプセル — {YYYY-MM-DD}

### 1 年前の今日 ({YYYY-MM-DD})
> {ノート本文の冒頭 200 文字}

({リアクション数 / リプライ数があれば})

### 2 年前の今日 ({YYYY-MM-DD})
> ...

---
今日もよい一日を。
```

該当が 0 件なら 1 行で:

```
✓ タイムカプセル: {YYYY-MM-DD} — N 年前の同じ日に投稿は見当たりませんでした。
```

## 制約と注意

- **本人のノートのみ**。他人の過去ノートを掘り起こさない (リプライ先も引用しない)
- CW 付き / 鍵 (followers / specified) のノートは「[CW: 理由]」だけ表示し、本文は出さない。元の公開範囲を尊重する
- 過去の自分が言ってることに対して **評価しない**。「黒歴史」「成長したね」みたいな茶化しは禁止。事実として並べるだけ
- 引用は冒頭 200 文字まで。長いノートは「...」で省略
- ネガティブな内容 (体調不良、葬儀、別れなど) でも検閲はしないが、末尾の挨拶 (「今日もよい一日を」) は付けない方が自然な場合は省略してよい
- アカウントが新しくて 1 年前のノートが存在しないなら、N=1 でその旨を出して終わる (N=2 以上を試す必要なし)
- リモートサーバーが落ちてる等で取得失敗したら 1 行で「過去ノートにアクセスできません」と告げて終わる
